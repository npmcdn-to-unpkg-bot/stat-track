As it stands, this app is built from three containers:

  • web_ui - This generates the web page.
  • web_api - This negotiates the api call.
  • db - This stores the dummy data.

I've used the alpine linux container to build a base container for the two web containers, but I've added python3 and flask to it. Each of the containers in the service are included in the docker-compose.yml and thus, the service can be started with a single terminal command. The database is a rethinkdb 2.3 from docker hub.

The next steps are to:

  • Load real data into the db.
  • Aggregate the data.
  • Display the data on the ui.
  • Get user input about what data to get.
  • Make it look nice.
